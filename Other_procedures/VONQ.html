The VONQ procedure

Whenever you receive a request from StSt.de to set-up a new end customer for VONQ, you should follow this procedure:

  • In Salesforce, the customer needs to be set up under end name of a company (by DE), with parent company as Reseller – VONQ, here is an example of one of the VONQ accounts:


  • DE sends a WW under the end customer (under Virtu Hire in this above example), and DE team sends us an email with all information to start the set up
  • TNW needs to create a contract in SF manually with the following new dates: (valid until 2026!!!)
    • Contract start date: 30/10/2025
    • Contract end date: 29/10/2026
    • Duration: 12 months
    • TNW should link the WW to the contract so the WW does not expire
  • TNW creates manually a customer in FP (with extension _VONQ_DE in the name), and delegates channels in FP manually (as per DE email details):


  • We need to make sure to tag 'VONQ' as a provider under account in FP (when creating FP customer):


  • TNW creates FP wallet manually as a 'customized' one:


  • In FP wallet we need to add the customer and the credits manually (credits always are added as GFP – 20 credits by default to everyone (20 is a safe number for 1 year ahead). As for the type, DE team in their original request should indicate this - the country and type of credits (standard or customized).
  • We need to send the FP customer ID to DE team: (for example, see below)


  • We need to inform the receiving partner(s) via an email, explaining VONQ set up and why they cannot see anything in SF yet. At the same time, we should confirm to DE that the delegation(s) is done, including the customer ID as per point above).
  • Example of an email to the receiving partner:

    • Dear Pnet Team,
    Please, note that this is a special set up for an end customer of VONQ, therefore you won’t be able to see their contract details in SF.
    Contract no: C062654.
    Contract start date: 2025/10/31
    Contract end date: 2026/10/30
    Their listings will be covered with monthly Pos at the following price:
    Product: Pnet | Standard Listing unit price rate - 15 % AE and - 1 % Reseller discount. (Comment: this information has to be provided in original DE request. If they missed providing it, ask them for more details)

    Please, let me know if you have any questions.
    Kind regards,
    The Network Team

  • Example of an email to DE:

    • Hi DE,
    Here is FP customer id for the provider: 4978905965928081177.
    The delegation email was sent out to Pnet team as well. Please, allow them few days to set it up in FP.
    Kind regards,
    The Network Team
