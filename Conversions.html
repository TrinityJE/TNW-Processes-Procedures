Conversion

Conversion used to be something offered only in exceptional cases, and it got more standardised during COVID period. However, in 2023 it was decided that conversion will become a normal procedure, and a whole process (and Salesforce developmnt) was created for it.


The new conversion procedure is as follows:

  • GFP conversion to local always allowed
  • GFP conversion to GFP conversion is allowed using the following rules*
  • Use Conversion.xlsx to figure out the credits that should be converted
  • Create CP(s) for the converted product(s) and use the CP attribute ‘Conversion’
    • This is important for reporting purposes
    • By using this attribute, the total gross & net amount of the CP will NOT add up to the total gross & net amount of the contract!!!
  • Issue the PO for the converted product(s)
  • Issue an ADMINISTRATIVE PO from the product that is being converted (should always be GFP):
    • Mark the PO as ‘Conversion PO’ (under the check box for ‘Negative PO’ when creating the PO header)
    • This is very important as credits in such a PO will not be counted into ‘consumed’ field of the CP but into newly created ‘converted’ field so we can distinguish the number of consumed credits vs number of converted credits!
  • Issue the PO to TNW
  • The number of credits should be the number of credits being converted
  • Once OUT PO is created, change the ‘You can invoice us €’ field into 0€
  • Check that the splits are 0/0/0

* You can only convert GFP credits into a higher, EXISTING level. For example, credits of L1 standard can be converted into L1 customized, and L2/3/4 standard or customized; L4 customized, however, can be only converted into local credits.
