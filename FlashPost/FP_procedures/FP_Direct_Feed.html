Import of direct feed credits into FlashPost

When there is a new quote including job credits with direct feed as posting method, we need to import these in FlashPost.

You should import the credits upon noticing a direct feed product in the quote when checking it. Soon, there will be an automated Slack notification that will let you know there is a direct feed that needs to be imported.

Import the contract in FlashPost via Customer tab > Import (it is basically the same process as for FP Regular credits). However, do NOT delegate the channels! Currently the system still offers delegation, but you need to uncheck the box and NOT delegate. Soon, this step will be automatically skipped when importing direct feed credits.

In case this is a new customer in FlashPost, you will have to create the customer account first. For that, you will need the logo (should be in SF as it is mandatory now, but there might be cases when older customers who always used direct feeds will need to get set up, so then ask the partner for the logo). As a posting method, choose Recruiter Application.

After import, what happens is:

  • the PO is created in Salesforce
  • the wallet is created and the credits are marked as “Direct consumption credits” in the wallet (or as “No FlashPost credits” in Customers > Wallet)

The credits are not available for posting via FlashPost.


When it comes to updating consumption, this is a new task for the receiving partner. If consumption for at least one direct feed product hasn't been updated in the last 30 days or more, the receiving partners will receive an email (on Mondays) asking them to update the consumption.
