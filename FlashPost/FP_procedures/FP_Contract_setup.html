Contract set-up and delegation - New, Renew, Extend


How to set up new channels for customers in FP?

  • Import the customer from Salesforce to FlashPost:
    • Confirm the products and channels that should be created
    • Add a logo and ENABLE the customer
    • Assign it to CS person in charge of the customer
    • Select the customer type - Self-Service or Regular and the posting method - Automated FP or Recruiter Application
      • If Automated FP, select the provider. By selecting the provider, the customer will be immediately added to the provider user
    • Save the changes
  • Verify the newly created wallet
  • Chase non-responsive partners (based on the summary email you receive twice a day)
  • If customized contract, send the template information in a separate email to all concerned partners (not all partners support customized) and for StepStone countries, created it in FP
  • If additional info is needed or requested, send it in a separate email to the concerned partner

How to set up renewals/extensions in FP?

  1. (Early) Renewal - 2 contracts running at the same time with the same product type(s) available per country
    • Delegation emails to be sent for the same channels without disabling them

  2. (Early) Renewal - 2 contracts running at the same time with the different product type(s) available per country
    • Add new channel types manually (with the help of “Job boards” list in the newly created wallet) and leave existing ones enabled
    • Delegate the renewal only to newly created channels
    • If channel with new product type is not added, below can happen and it can cause issues on fully-automated partners such as AT, BE, DE and NL:


    • When both channel types are available for one country, the client will be able to indicate listing type when creating a listing


    • Once the old contract expires, old channels should be deactivated as well (hard to follow-up on though)
    • Reason for this is that the client will still be able to select between the two types, even if they have only one type of credits available for that country
    • Certain partners (a.k.a STST) will report an error if product type is incorrect and it doesn’t match with what they setup in the back-office. This can happen even if channel was confirmed, so the issue might not be very obvious at first
    • Check the incoming xml file to make sure your assumptions are correct. If product type doesn’t match with what is available in the wallet, this could be the reason for the error




    • Still double-check with the partner and confirm what product type they received from us
    • Proceed with disabling the incorrect channel type and repost the listing as new (do not use the repost button!!!)
    • Now the system will no longer give the user the option to select the product type
    • If you try to repost the job in error (even if you disabled the old/incorrect channel) you will get PERMISSION ERROR reported by Flashpost:


    • If the job board still reports an error after you made the necessary corrections, issue should really be coming from setup they did in their back-office

  3. Renewal - after previous contract expired, same product type per country
    • Delegate the renewal and at the same time disable all relevant channels



  4. Renewal - after previous contract expired, different product type per country
    • Make sure to disable channels with wrong product type and at the same time manually add new ones (again help yourself with “Job boards” list in the in the newly created wallet)
    • Delegate renewal only for newly created channels
