Converting a customer already existing in FP into a 'Self-Service' one

You might come across a situation when the outbound partner needs to use the self-service feature but is not able to create the customer because that customer is already existing in FlashPost (it used to be FP customer in the past and/or they had postings to Appcast, Infojobs, or Daijob).

In order to import the credits for such a customer, you need to choose the correct scenario and follow up the steps:


IMPORTANT: If the existing FP customer has a running set-up (a running wallet for other countries than Infojobs, Appcast, and/or Daijob), they should keep using the set-up and they should NOT be switched into self-service customer. Any new purchase for such a customer should be treated via the normal set-up and delegation process.

IMPORTANT: We do describe the procedure for GFP credits below as well. However, GFP packages should use the regular set-up (FPRA or AFP) and only be used for self-service as an exception!


For SCENARIO A:

Customer has a running contract with FP set-up for Infojobs, Appcast, and/or Daijob and there is a new CP for job listings (local and /or GFP) within the SAME contract.

In such case, it is possible for the partner to import LOCAL credits for such customer; GFP credits have to be imported by admin

Follow these steps:

  1. Switch the customer into 'Self-Service'
  2. In case of local credits:
    • Switch the wallet type into 'Self-Service'
    • Ensure that the 'posting method' on the relevant CP(s) is 'Self-Service' (if more than 10 jobs, you need to change this as admin)
    • Ask the partner to import
  3. In case of GFP credits:
    • Go to channels and verify what channels should be used:
      • Delete those that the customer will not need using the delete button
      • Change the rest into email override in case of a fully-automated partner
      • Add new channels as email override if necessary
    • Change the wallet type into 'Salesforce imported and monthly PO enabled' so monthly PO can be created
    • Go to wallet and import the CP(s)
  4. In case if there is a mix of local and GFP credits, follow up the procedure for GFP credits + ask the partner to create the upfront PO for local jobs

For SCENARIO B:

Customer has a running contract with FP set-up for Infojobs, Appcast, and/or Daijob and there is a partially used CP for job listings (local and/or GFP) within the SAME contract

In such case it is not possible for the partner to import the credits for such customer

Follow these steps:

  1. Switch the customer into 'Self-Service'
  2. In case of local credits:
    • Go to channels and verify what channels should be used:
      • Delete those that the customer will not need using the delete button
      • Change the rest into email override in case of a fully-automated partner
      • Add new channels as email override if necessary
    • Go to wallet and import the CP(s)
    • Change the wallet type from 'Salesforce imported' into 'Self-Service' so the wallet is ready for further import should there be an upsell on this contract
    • Ask the partner to create an upfront PO for the remaining credits
  3. In case of GFP credits:
    • Go to channels and verify what channels should be used:
      • Delete those that the customer will not need using the delete button
      • Change the rest into email override in case of a fully-automated partner
      • Add new channels as email override if necessary
    • Go to wallet and import the CP(s)
    • Change the wallet type into 'Salesforce imported and monthly PO enabled' so monthly PO can be created
  4. In case if there is a mix of local and GFP credits, follow up the procedure for GFP credits + ask the partner to create the upfront PO for local jobs

For SCENARIO C:

The customer exists in FlashPost but doesn't have any active contract in FlashPost (i.e. no active wallets) and they have signed a new contract (for local and/or GFP listings) which wasn't consumed (even partially) yet.

In such case, it is possible for the partner to import LOCAL credits for such customer; GFP credits have to be imported by admin.

Follow these steps:

  1. Switch the customer into 'self-service'
  2. In case of local credits:
    • Ensure that the 'posting method' on the relevant CP(s) is 'Self-Service' (if more than 10 jobs, you need to change this as admin)
    • Ask the partner to import
  3. In case of GFP credits:
    • Go to channels and verify what channels should be used:
      • Delete those that the customer will not need using the delete button
      • Change the rest into email override in case of a fully-automated partner
      • Add new channels as email override if necessary
    • Go to wallet and import the CP(s)
    • The wallet type should be 'Salesforce imported and monthly PO enabled' so monthly PO can be created
  4. In case if there is a mix of local and GFP credits, follow up the procedure for GFP credits + ask the partner to create the upfront PO for local jobs
  5. In case if there is a mix of local and GFP credits, follow up the procedure for GFP credits + ask the partner to create the upfront PO for local jobs

For SCENARIO D:

The customer exists in FlashPost but doesn't have any active contract in FlashPost (i.e. no active wallets) and they have signed a contract (for local and/or GFP listings) which was already prtially consumed

In such case, it is not possible for the partner to import the credits (for the artially used CP) for such customer

Follow these steps:

      • Add new channels as email override if necessary

    • The wallet type should be 'Salesforce imported and monthly PO enabled' so monthly PO can be created
