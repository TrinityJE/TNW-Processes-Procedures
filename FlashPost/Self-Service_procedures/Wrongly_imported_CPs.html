Wrongly created CPs imported into FlashPost

If the partner makes a mistake and creates a wrong Contract-Product that is then imported into FlashPost, you should follow below procedures for different scenarios:


The contract type is SJNWW (product type/price not covered by SJNWW)

If listing was not posted:

  • Revoke it and inform partners
  • Subtract imported credit(s) from the wallet
    • To do that you need to copy the CP ID from Salesforce (the link of the page)
  • Delete the PO created upon the import
  • Delete the incorrect CP but remember:
    • You need to un-tag the box "CP imported by FlashPost" otherwise you will not be able to delete the CP
  • Delete SJNWW contract after partner’s approval
  • Ask the partner to create a type “New/Renew” contract with correct CP
  • Ask the partner to import credits & repost the job
  • Validate the new PO

If listing was posted:

  • No changes needed in FP
  • Delete the PO created upon the import
  • Delete the incorrect CP but remember:
    • You need to un-tag the box "CP imported by FlashPost" otherwise you will not be able to delete the CP
  • Delete SJNWW contract after partner’s approval
  • Ask the partner to create a type “New/Renew” contract with correct CP and PO
  • Validate the PO and inform the RP about new PO for the already posted job

The contract type is SJNWW (product type/price covered by SJNWW)

Wrong product type/price (could be that prices were not updated with Q update)

  • No changes needed in FP
  • Delete the POLI
  • Correct the CP by changing product type or updating unit price directly in the country itself
  • Add the POLI back into the PO and approve it

The contract type is New/Renew

Wrong country:

  • Revoke the listing if it wasn’t posted yet and inform partners or expire and refund in case it was posted
  • Subtract imported credit(s) from the wallet
    • To do that you need to copy the CP ID from Salesforce (the link of the page)
  • Delete the IN and OUT PO created upon the import
  • Delete the CP and create a new, correct one (for this, ensure that the correct country is in the contract and that there is an available WW)
  • Ask the partner to import credits & repost the job
  • Approve the correct PO

Wrong product type/price/discount:

  • No changes needed in Flashpost (wrong product type not an issue when Self-Service)No changes needed in FP
  • If PO approved, delete OUT PO and put IN PO on “To Validate”
  • Delete the IN POLI
  • Amend the incorrect info in the CP (you will need to untagged the ‘used in PO’ box first)
  • Once CP is corrected and save, edit it again by tagging the ‘used in PO’ box
  • Add the correct IN POLI, and validate the PO
  • Inform partners if new OUT PO
