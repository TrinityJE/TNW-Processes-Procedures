Yearly Invoices

The Network is issuing a yearly invoice to all its partners. This invoice is based on report of active and received sales for the given year that should be pre-run in December but the final update should happen at the beginning of January to have the full view of the previous year.

The most important points connected to yearly invoices are:

  • Issued in the first 2 months of the given year (Jan - Feb)
  • They comprise of Salesforce license fee + Marketing Budget (partnership fee)
  • If partner's active and received sales are less than 30k, they are recharged half of SF yearly fee
  • In our T&Cs:

    Partner shall pay 950 USD per User License per year for the use of the CRM. Each Partner purchases at least one User License. That price can evolve in time because of salesforce pricing changes or exchange rate evolution between Euros and USD. The yearly fee shall be invoiced upfront at the beginning of each year and shall be payable within 30 days.

  • Partnership fee is 750 EUR in the 1st year and 1500 EUR 2nd year and after, as per details in our T&Cs:

    Year one of Partnership: 750 €, if a Partners joins the Network before July, the Network will invoice 750 € during that year. If a Partner joins the Network in or after July, the Network will invoice these 750 € in January of the following year.

  • StSt Group, JobCloud and Figaro are our shareholders, paying higher partnership fee + shareholder fee, as per details in our T&Cs:
    1. StepStone Group EMEA: 185.000 €
    2. Figaro Classifieds A.S.: 105.000 €
    3. Jobcloud AG: 105.000 €

The Yearly Invoice Procedure is as follows:

1. In December, go to the folder Invoices from Network to Partners and:

  • Go to the current year folder, Yearly invoices subfolder, and open the ‘xxxx yearly invoices’ excel file
  • Save the file as a copy
  • Go back to Invoices from Network to Partners and create a new folder for the following year, subfolder Yearly invoices
  • Save the copy in here and rename it into ‘xxxx yearly invoices’ (i.e., 2023 yearly invoices) + remember to rename all the tabs into the current year

2. For the pre-liminary check, open the file and work with the following tabs:

  • Budget information = a list of partners with the final invoice amount to invoice / what we are invoicing them for / notes if we should invoice them or not, if we should include some fees or not, etc. There are few partners we do not invoice as they invoiced us for less than 5K in the previous year; they could leave the alliance (usually marked in yellow)
  • CRM Users data = info about SF users
  • CRM Users xxxx = final data on SF users + calculation on the number of SF users per partner
  • All in xxxx = the main tab where we are gathering all the info and doing the actual calculations
    • We can find here ALL partners, even those that we do not invoice or that we do not work with anymore (majority are now hidden), including our shareholders JobCloud + StepStone Group + Figaro
    • Here we decide if a partner should have a full / half / or none recharge (the general rule is = if the sum of Sales is more than 30.000€, then full recharge, if it is less than 30.000€, then half recharge + some partners (less then 5.000€ as per above) are not recharge at all
  • Partners Values (hidden) = data about partner's active and received sales

3. First, we start with the check of CRM users for the given year:

  • Go to Salesforce and run Active users of Salesforce report
  • Keep the filters as they are and export the file
  • Merge first and last name for the purpose of our pre-liminary report; simply take the first two columns and copy paste them into the tab ‘CRM Users data’ (we need to unhide the tab); there is already a formula which will create the merged name for us (pay attention that you override all the data from last year – if there are less users this year!)
  • Then, simply copy paste the rest of the information (profile, company name, username, active, last login, and created date)
    • Column G (active) might give us an error as it is not a number – simply click on the error and say you want to convert it to a number
    • For columns H (last login) and I (create date), change the format to short date
  • Sense check that each entry has a correct company name (correct if you see some discrepancies and synchronize the naming)
  • Once the data in the tab ‘CRM Users data’ is correct, we simply take it and copy paste it into the tab ‘CRM Users xxxx’ – careful when coping, double check that NO old data stayed in
  • In the tab 'CRM Users xxx', refresh the pivot table

4. Now we move to the tab 'Partners values' to get the accurate sales data (we need to unhide the tab):

  • Ask the Power BI expert to either insert the data for active & received sales or do it yourself if provided in excel format

5. Next, go to the main tab ‘All in xxxx’ and do the final sense check and correct everything that needs to be corrected:

  • The column Outbound sales is looking at Partners value tab, Active Sales xxxx part – if there is -€, double-check in the Atvie Sales xxxx table if there is no sale (Remember! Sometimes the partners name can be wrong – in that case we correct the name and the number updates accordingly)
  • The column Inbound sales is looking at Partners value tab, Recieved Sales xxxx part - if there is -€, double-check in the Received Sales xxxx table if there is nothing received (Remember! Sometimes the partners name can be wrong – in that case we correct the name and the number updates accordingly)
  • Nb users column is a vlookup that checks the total number of CRM users per partner – if there is 0, double-check in the CRM Users tab if there are no active users (Remember! Sometimes the partners name can be wrong – in that case we correct the name and the number updates accordingly)
  • Recharge type – check if the partner should have full / half / none recharge as per the rules above and update accordingly
  • Mark in red all the partners that we will not recharge

6. At the end, we update the ‘Budget infortmation’ tab to have the final overview

7. In January, go to this excel file and update the following information:

  • ‘CRM Users data’ and ‘CRM Users xxxx’ tabs (with SF report)
  • ‘Partners values' tabs (with Power BI report)
  • Final check on ‘All in xxxx’ tab
    • By January, we should also receive a recharger invoice from StepStone for SF users - based on the total amount, we might need to adjust the price per license (or to decide to absorb the cost if necessary); the balance 'The Network absorbs' should usually be 0€

How to issue the yearly invoices?

  • In finance@the-network.com, look up the last year 'Yearly Invoices' email with all instructions
  • Check and update the information (inlcuding information about new partners and company details, exceptions, etc.).
  • Attached the new file and send it to the accountant
  • Once the accountant has it ready, we check each individual invoice and send it to the partners
  • The partners have 30 days to pay
