Under consumption passthrough

This process is to be done every 6 months. The finance part, which was done last in August 2022, it was for the contracts which ended until the end of December 2021. (Always contract end dates). A new report and POs to be created in September 2022 for the contracts end dates from 01/01/2022 to 30/06/2022, etc.

The process should always consider the renewal 2 months period when partners have up to 2 months to renew an expired contract (a period we need to consider in case there are carryovers, extensions, etc. That is why we do not do it in June/January but rather in September/April.

To perfrom a successful under consumption procedure, you should follow these steps:


Sales Process

  • Go to the pool where all files for underconsumption are stored
  • Create a new folder for the period you are going to work on (i.e., 20220101 until 20200630)
  • To know which contracts are concerned for the given period, you need to run a SF report ‘Underconsumption 10% NW’
  • Change the date filters: contracts which expired from 01/01/20xx till 30/06/20xx / from 01/07/20xx till 31/12/20xx (as we are doing it every 6 months). Remember! The exercise should be only done min. 2 months after the filter end date (September / April).
  • Export the report into xls, details only
  • Tidy up the report:
    • Save the report into the folder you just created and call it '10% underconsumption all 20xxxxxx to 20xxxxxx'
    • Create an actual table from the data
    • Rename the columns to be tidier
    • Change the Net Price column in actual currency (€)
    • Move the column Partner in charge from the last place to the first place (column A)
    • Rename the sheet into "Data"
  • Add several new columns:
    • Column M 'Net unit price'; you need to add a formula: Net price eur (L) / No of jobs (K) --> =[@[Net price EUR]]/[@[Nb Jobs/Product]]
    • Column O 'Under-consumed'; you need to add a formula: No of jobs (K) - Consumed (N) --> =[@[Nb Jobs/Product]]-[@Consumed]
    • Column P 'Adjustment'; we should only enter a negative number here
    • Column Q 'Adjustment reason'; we should state a reason why we entered a negative number into Column 0
    • Column R 'Under-consumed Final'; you need to add a formula: Under-consumed (O) + Adjustment (P) --> =[@[Under-Consumed]]+[@Adjustment] (the result should be negative)
    • Column S 'Passthrough %'; per default, we add 10% for everyone. However, we need to adjust the split if case of intra-StepStone consumption on LOCAL JOBS only (GFP is automatically 10%). By using filters on Partner in charge, we check StepStone Group countries (Careerjunction, Carribean jobs, Irishjobs, Jobs.ie, Jobs.lu, Nijobs, StSt AT, StSt BE, StSt DE, StSt NL, Tecoloco, TJ) and:
      • If they posted to a StepStone country, we change the split to 0%
      • If they posted to a non-StepStone country, we keep the split 10%
      • If they posted to The Network (translation), it is up to our judgement if we want to invoice them for this or not
    • Column S 'The Network invoices'; you need to add a formula: (Under-consumed final (R)) * (Net Unit Price (M)) * (Passthrough % (S)) --> =[@[Under-consumed Final]]*([@[Net Unit Price]]*[@[Passthrough %]])


  • Add two new sheets with pivot tables to show the final data clearly:
    • Overview partners - select Partner in charge in rows, and 'Sum of The Network invoices' in values
    • Overview customers - select 'Partner in charge' and 'Customer' in the rows, and 'Sum of Under-consumed Final' and 'Sum of The Network invoices' in values

  • Back in Salesforce, run another report ‘Negative POs by contract end date’ and change the filters of contract end date to the same dates: from 01/01/20xx till 30/06/20xx / from 01/07/20xx till 31/12/20xx
  • Export the report into xls, details only
  • Open this new report in a different window for better visualization
  • Using the report of negative POs, we match CPs with the 10% underconsumption report we just saved (we can use Vlookup function or do it manually)
  • When we find a match (there IS a negative PO for a CP that is on our 10% underconsumption report), we add this information into the column P (Adjustment) using a negative number and at the same time we add a reason to column Q (Adjustment reason), stating there was a negative PO (and we can add the negative PO reference). Any other records can be ignored as they are not in our 10% underconsumption report.

  • Once more we go back to Salesforce and we run the last report 'Carry and roll overs for 10% underconsum' and change the filters of contract creation date; for example starts from 01/11/20xx for the period 01/01/20xx till 30/06/20xx or 01/05/20xx for the period 01/07/20xx till 31/12/20xx (we look at CP creations dates as of the first day when the under consumed contracts could have expired and we go 2 months behind just in case there were some corrections)
  • We manually check each entry (we can skip a lot of entries as they have 'roll' in the comment used as a job title) but we mainly focus on GFPs
  • If we find something relevant, we again update column P (Adjustment) and column Q (Adjustment reason) as we did when checking the negative POs report

  • Return back to the 10% underconsumption report and focus on the three comments fields ('Contract comment', 'Admin contract comments', and 'CP comments') to eliminate another potential CPs that should not be charged
  • If we find something relevant, we again update column P (Adjustment) and column Q (Adjustment reason) as we did when checking the negative POs and Carry and roll overs reports
  • After we do the necessary changes, we delete these threee colums ('Contract comment', 'Admin contract comments', and 'CP comments') as they will not be relevant for the partners

  • We save a copy of the final report, and from the copy, we start generation reports for each partner
  • The moment we start working on a new report, we save it immediately - the name stays the same (remove COPY reference) but we include the partners name before the date (i.e. 10% underconsumption Appcast 20220101 to 20220630)
  • In each report, we keep the 'Customer overview' and 'Data' sheets and we delete the 'Overview partners' sheet
  • In the 'Data' sheet, we order the data by 'Partner in charge' and simply delete all unnecessary entries (i.e., if we are doing a report for Appcast, we keep Appcast records but delete everything else)
  • Once we deleted all unnecessary data, we update the 'Customer overview' sheet which will show us the total that we should charge amount the partner for

  • We then communicate the amount and the CPs to partners to have their feedback on potential corrections (negative POs, carry overs, etc.). All the communications are in the Sales mailbox - an old email should be reused, and we should simply change the dates and the amount to invoice and leave the rest of the info as it is


  • Create a new folder ‘Final’ - here, we should put all the partner reports once confirmed (if necessary, we first do the unnecessary in column 0 with a minus number and we update the pivot table)

CS Process

  • After the deadline, once we are sure about the values, we start issuing the actual under-consumed POs in Salesforce
  • We go to the contract that was under-consumed and we check:
    • The Contract-Products to see which one is under-consumed; there might be more than one (keep this info in mind as you will need it when creating the WW for The-Network)
    • The Contract-Countries to see if the country The Network.1 is included
    • If yes, we check if there is WW linked to the contract and if yes, we need to edit it and tag the 'never expire' field
  • If the C-C or the WW are not linked to the contract, we open the customer in a seperated tab and check if there is an existing WW for The-Network
    • If yes, we clone it (take the latest one) & answer it as Done (in case there are several under-consumed CPs, we need to mark it as 'never expire' as well!!!)
    • If no, we create new Account-Country, The Network.1 (it does not matter what you put as info into centralization, etc.), and answer it as Done (in case there are several under-consumed CPs, we need to mark it as 'never expire' as well!!!)
  • Remember: In both cases, you should add a comment to the WW indicating this is for under-consumption excercise, ex: ‘Invoicing of 10% of under consumed credits for 01/01/2022 - 30/06/2022 exercise.’

  • We then go back to the under-consumed contract & add The Network as Contract Country (unless it alraedy exists)
  • Attention: In case the under-consumed contract was a SJNWW, we have to create a new contract, or we can use some other existing contract instead
  • We create a new C-P:
    • Product - 'Under Consumption Pass-Through'
    • Country – The Network.1
    • Receiving partner: The-Network
    • Number of products: 1
    • Total value – taken from the partner final file (i.e., PO248595)
    • Add the following comment, ex: ‘Invoicing of 10% of under consumed credits for 01/01/2022 - 30/06/2022 exercise.’
  • In case there were several under-consumed CPs under the same contract, we create a new CP for each entry
  • We create an ad-hoc PO (in case of several CPs, we can consolidate all of them in ONE single PO) with 1 month duration:
    • We mark a check box: Under-consumption PO
    • We add POLI with the same comment as per above
    • We submit and validate the PO
    • We open the OUT PO, and we mark a check box as ‘Under-consumption PO’ manually
    • We validate the OUT PO

  • Once we have all POs done and validated, then we need to check that all is correct and we check it in the following report ‘Underconsumption POs created check’
  • We match the total of this report with the total of POs in xls sheet (for IN_POs and OUT_POs)
  • Once these numbers match, then underconsumption process is complete and Finance team is informed, so that these new numbers could be included in the next closing

Finance Process

  • The closing is done in the same way as always. After we have completed the closing files and all numbers are correct, then we are issuing underconsumption xls file as a separate doc for Bastian (ex, it will be called: ‘20220731_Underconsumption POs.xlsx’)
  • From the final closing file (20220731_Invoice information with Total Net.xlsx), we create a copy of it with the following modifications: we hide ‘Stepstone to Network’, Network to StepStone’, Invoices NW to Partners’
    • We unhide a new sheet called: ‘Invoices Under Consumption’ + ‘Report’. And we call it ‘20220731_Underconsumption POs.xlsx’
    • In the pivot filters of the table, we choose Under Consumption PO: 1
    • In the pivot filters of the table, we choose Purchase Order: record type: In
    • We re-refresh the table and save the file as ‘20220731_Underconsumption POs.xlsx’
  • We send this file to Bastian, informing him that he needs to issue additional invoices to our partners. Once we receive them, we send them to our partners
