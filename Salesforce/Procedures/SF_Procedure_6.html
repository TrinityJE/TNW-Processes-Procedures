Modification of WW in case of (de)centralization

For now, we don’t have an automated solution for situation when a WW changes from centralized to decentralized or vice versa.

If such a situation arises, follow these steps:

  1. Single valid WW (or a handful of valid WWs), no protection
    • Ensure that you have a confirmation from the inbound partner that the change can be done
    • Edit the WW manually
    • Change the info about (de)centralization
    • Put a comment with TNW amendments
    • Save
    • Attach the related email

  2. Single valid WW (or a handful of valid WWs), protection
    • Ensure that you have a confirmation from the inbound partner that the change can be done
    • Edit the WW manually
    • Change the status into ‘To Do’
    • Change the info about (de)centralization
    • Put a comment with TNW amendments
    • Save
    • Go to the WW product and change the value in the WW product into 0.01€ (you can keep the original value in the comment field); this only applies for cases when the WW is changed from centralized to decentralized
    • Put the WW back ‘Waiting for approval’
    • Attach the related email

  3. More valid WWs
    • Mass clone the WWs

  4. Expired WW(s)
    • Clone the WW (mass clone the WWs)
