Procedure for creating an upfront PO from a GFP package

In exceptional cases, it is possible to create an upfront PO from a GFP package to block certain credits/amount for the receiving partner.

Always warn the selling partner that giving an upfront PO from a GFP package can have as a consequence loss of credits (in case the customer does not post the number of jobs that were given via the upfront PO, they will lose those).

To create an upfront PO from a GFP package, please follow this procedure:


UPFRONT PO for covering protection:

Scenario 1: the upfront PO was requested BEFORE the CP is imported in FP (the ideal scenario)

  • Create the upfront PO for the receiving partner
  • Import the CP -> all the credits will be imported, but the upfront PO credits will be ignored for monthly PO creation
  • It is during the import itself that the system will suggest creating an upfront set-up to make sure the upfront credits are reserved:
    • Copy the CP reference
    • Go to Admin Console -> Credits -> Set upfront GPFs
    • Here, insert the CP reference in the Contract Product Name field, select the concerned receiving job board, and the number of upfront credits
    • The system will reserve the credits for the receiver (with no 80% rule limit)
  • Then, in the wallet, in the distribution tab, you will see there is a new entry for upfront GFP with the reserved credits
  • You do have the right to undo it in case the upfront PO get cancelled -> simply repeat the same steps in the Admin Console, but imput 0 into the Upfront credits section

Scenario 2: the upfront PO was requested AFTER the CP was already imported in FP

  • The difference with Scenario 1 is that you do NOT import anything in FP anymore
  • Simply follow the process above from the Admin Console part on


Upfront PO for DIRECT ACCESS:

When creating an upfront PO from GFP to have the credits used via direct posting method (such as Strabag), the process is different and there is only one scenario possible, as you must import the CP:

  • Imprt the full CP into FlashPost
  • Manually deduct the number of credits in the FP wallet
  • Create a manual upfront PO for the number of credits that should be used for direct poting to the receiving partner
